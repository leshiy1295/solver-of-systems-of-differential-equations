СПРАВКА ПО ИСПОЛЬЗОВАНИЮ
Данная программа предназначена для решения систем дифференциальных уравнений в нормальной форме.

Входными данными являются:
-разрядность системы, т.е. количество уравнений в системе,
-метод интегрирования (Эйлера, Рунге-Кутта 2-го или 4-го порядка),
-имя переменной-аргумента,
-собственно система дифференциальных уравнений в нормальной форме,
-границы отрезка интегрирования,
-значение аргумента в условии задачи Коши,
-количество точек для интегрирования на отрезке,
-условия задачи Коши,
-дополнительные переменные.

Выходными данными являются:
-журнал работы программы (файл log.txt),
-временный файл temp, который можно сохранить в качестве отдельного файла по желанию пользователя, содержащий отчёт о входных данных и выходных данных в виде таблицы значений,
-графическое представление результатов.

Для того, чтобы перейти к вводу системы уравнений, необходимо нажать на кнопку "Новый анализ" в главном меню. 
После этого необходимо выполнить следующие шаги:
Шаг 1. Ввод разрядности системы - натуральное число.
Шаг 2. Выбор одного из трёх предлагаемых методов решения системы.
Шаг 3. Ввод имени переменной-аргумента системы - имя начинается с буквы, может содержать в себе цифры или буквы латинского алфавита, не превышает 20 символов.
Шаг 4. Ввод системы дифференциальных уравнений в строковом формате без пробелов. Ввод границ интервала - вещественные числа, ввод значения аргумента в задаче Коши - вещественное число, количество точек на отрезке - натуральное число, включающее в себя точку, использующуюся в условиях задачи Коши, и значения функции на отрезках интегрирования.
Также возможно указать дополнительные переменные с помощью окна дополнительных переменных, открывающееся по нажатию соответствующей кнопки.
Шаг 5. Ввод задачи Коши с возможностью дополнительного указания, какие именно графики на следующем этапе необходимо вывести на экран.
Шаг 6. Если все параметры формально введены верно, то после начала решения окно дополнительных переменных закроется, если оно было открыто, а основное окно программы станет неактивным на время решения системы. После этого пользователю будут выведены требуемые графики функций-решений системы, а также появится кнопка "Сохранить результаты", позволяющая сохранить отчёт по входным-выходным данным системы.
После этого можно галочками скрыть/показать определённые графики, изменять входные данные для повторного решения системы.

Выражения для системы необходимо вводить без пробелов, используя арифметические операции, скобки, имена других переменных, объявленных в окне дополнительных переменных, переменных системы, а также математические функции:
sin(double) - синус,
cos(double) - косинус,
tg(double) - тангенс,
ctg(double) - котангенс,
arcsin(double) - арксинус,
arccos(double) - арккосинус,
arctg(double) - арктангенс,
arcctg(double) - арккотангенс,
log(double;double) - логарифм по основанию <double> от <double>,
lg(double) - десятичный логарифм,
ln(double) - натуральный логарифм,
pi() - число пи,
e() - число e,
sqrt(double) - корень квадратный,
sqr(double) - квадрат,
pow(double;double) - double в степени double,
exp(double) - e в степени,
fact(int) - факториал числа.

При вводе имён функций и переменных обязательно необходимо учитывать регистр. Буквы не латинского алфавита не допустимы.

В окне дополнительных переменных можно указать дополнительные переменные системы, которые можно использовать в качестве:
-переменных для подсчёта каких-то значений вместо использования калькулятора,
-вычисления значений систем выражений (переменные могут зависеть от других переменых => программа обнаруживает ошибки и вычисляет значения всех переменных сразу же после их добавления)
-использования в качестве параметров в системе дифференциальных уравнений.

На имена переменных существуют следующие ограничения:
-они должны быть синтаксически корректны (начинается с буквы латинского алфавита, далее - цифры или буквы латинского алфавита)
-они не должны совпадать (учитывая регистр) с именам стандартных функций, перечисленных выше,
-они должны быть уникальными (т.е. не может быть двух переменных с одинаковыми именами)
-они не могут иметь имена, используемые при реализации того или иного метода:
 при использовании всех методов: df<номер уравнения>, f<номер уравнения>, zakon<номер уравнения>, <имя переменной-аргумента>, h
 при использовании метода Рунге-Кутта II порядка также запрещёнными именами являются: k1<номер уравнения>,k2<номер уравнения>
 при использовании метода Рунге-Кутта IV порядка также запрещёнными именами являются: k1<номер уравнения>,k2<номер уравнения>,k3<номер уравнения>,k4<номер уравнения>
